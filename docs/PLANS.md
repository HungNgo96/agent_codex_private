# Plans

Use this file as the entry point for execution planning.

## Active Plans

- `exec-plans/active/`: work currently being implemented.
- Active plans are part of the primary harness-checked plan lifecycle.

## Completed Plans

- `exec-plans/completed/`: implemented and verified work.
- Completed plans are part of the primary harness-checked plan lifecycle.
- `superpowers/plans/`: historical Superpowers-format implementation plans kept for compatibility.

## Tech Debt

- `exec-plans/tech-debt-tracker.md`: known follow-up work not assigned to an active plan.
